5年目突入のMacBook Pro、昨今のAIブームで様々なツールを試し・失敗・再インストールを繰り返す。よせばいいのに、気になったAsahi Linuxなんて入れてみたりもした。
おかげで色んなツールが勝手にいろんなところ（しかも見えない `.xxx`フォルダーにとか）にインストールされ、ファイルがゴミと化す。

で、あてどなく「ディスクの容量が足りなくなってきます」とかのアラートが頻繁に出てくるようなった。調べてみると、どっかに2GBのファイル（LLM？）が鎮座していたりしてたりする。デスクトップはもう隙間がなくスクショや細かなファイルで占領される。捨てど捨てど、綺麗にならず。。グジャグジャかつカツカツな状況。

ちゅうことで、俺流メジャーアップグレードこと再再再再再クリーンインストールをしようと思い、今回は今までのマニュアル＆アナログ復元方式から、スマート＆ストイック復元方式に切り替えようと、AIに聞きまくって実験・整理中。

**ストレスの素**

- Hugoでサイトを作っていて、アップデートで構造が変わりビルドでエラー。。修正最中に、前のバージョンでビルドすれば済むんじゃねと、気がつく。
- このToolはこのバージョンを使って、あっちはこのバージョンだと動かないで沼へ。
- 久しぶりにToolを使おうと思って立ち上げたら、エラ〜〜あぁ〜〜。
- gem地獄、pip地獄、nodejs地獄・・・の地獄めぐり。。
- 上記が積み重なると、ストレスが倍増し、もうえ〜わ、やめ、となる。

で、とりあえずまとまった形：macOSを3つの層（最近マイブームのLayer分け）で考えた。

1. OS Layer：macOS + Command Line Tool + Podman（コンテナはここ）
2. コマンドラインツールLayer：Nix Packageで管理（ここんとこがカオス）
3. アプリケーションLayer：Brewもしくは直接落として管理

このかたち（NixPackageのおかげ）でディスク内に入り込んで無駄になったライブラリなどの整理・管理がしやすくなる。複数のバージョンをToolによって使い分けができるので、Toolをアップグレードした時に不具合が出ても、Tool内でのロールバックも可能。ある意味コマンドToolのサンドボックス化かな？

Nix Packageは初期設定と使い方を学ぶのに、学習曲線がちときついのだけれど、理解すれば合理的な選択で作られたもの（のよう）なので、しっくりする。管理もそこまで複雑ではなく、スッキリ度がかなり大きい！と期待、というかこれで最後にしたい。。

これはLinux（ちょっとMacからの移行も本気で考えてる）移行時にも使えるので非常に有益。（Windowsは知らん。）
