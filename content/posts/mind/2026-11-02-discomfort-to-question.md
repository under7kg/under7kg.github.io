> 違和感は、記事が間違っている証拠ではありません。  
> しかし、立ち止まって確認するための優れたセンサーになります。

ニュースを見ていて、「うまく言えないけど、なんかモヤモヤする。」と感じることが増えてませんか。まさにこの違和感こそがクリティカルシンキングの入り口です。

そういうとき、多くの人はこう思います。「ちゃんと言葉にしてから、整理してから考えなきゃ」と。でも実は、それが一番もったいない。**モヤモヤは、モヤモヤのまま扱ったほうが、本当の正体にたどり着けるんです**。

そして、その相棒にうってつけなのがAIです。この記事では、難しいプロンプト（AIへの指示文）の書き方なんて覚えなくていい、「生の違和感を、そのまま投げる」だけのシンプルなやり方をご紹介します。漠然とした「モヤモヤ」を言葉にして構造化するだけで、メディアリテラシーは劇的に磨かれます。

---

## 基本の考え方：整えるより、生のまま渡す

AIを使い慣れていない人ほど、「立派な質問を書かなきゃ」と気負ってしまいがちです。でも、きれいに整った質問は、聞く前に自分で答えを決めてしまっているようなもの。返ってくる答えも、当たり障りのないものになりがちです。

逆に、「なんか嫌だ」「どうも馴染めない」という生の感覚をそのまま渡すと、AIはその奥にある構造を本気で掘りにいってくれます。あなたの中のモヤモヤを、AIが代わりに言語化してくれるイメージです。

やることは、たった3ステップ。

1. 感じたことを、飾らずそのまま書く
2. そこに「考える力のキーワード」を数語ふりかける
3. 記事のURLを貼って、投げる

これだけです。

## 味付けに使う「クリティカル・シンキングのキーワード集」

生の違和感に、ちょっとだけスパイスを足す。それが下のキーワードたちです。丸暗記する必要はありません。「今日はこれとこれを使ってみようかな」くらいの感覚で、料理の調味料のように振りかけてください。

**思考を深める側のキーワード**
- 「合理的に批判して」——感情論ではなく、筋道で切ってもらう
- 「構造的な欠陥はある？」——論理の組み立て自体の穴を探す
- 「誘導されてない？」——読者を特定の結論へ導く仕掛けを暴く
- 「隠れた前提は？」——書かれていないけど当然視されている考え
- 「因果と相関を混同してない？」——「関係がある」を「原因だ」とすり替える手口
- 「分母は？比較対象は？」——数字のトリックをあぶり出す
- 「誰が得する構造？」——情報を出す側の動機・利害を探る
- 「情報源は誰？」——「専門家」「関係者」の正体をぼかしていないか

**自分にブレーキをかけるキーワード**（これが超重要）
- 「私が過剰反応してる可能性は？」——AIに同調・追従させないための一言
- 「逆の立場ならどう言う？」——反対側の言い分も出してもらう
- 「これは記事の問題？私の思い込み？」——違和感の出どころを切り分ける

最後のブレーキ系を入れないと、AIは全力であなたに賛成して「そうですね、けしからんですね」と煽り返してくることがあります。**自分へのツッコミを一行足す**——これがクリティカル・シンキングの肝です。

🌾：自分は検閲のないLLMを使っているので、**自分にブレーキをかけるキーワード**は必要ないのですが、ChatGPTなどのアプリでは「波風が立たないように？」調整されているみたいですね。

## 実例で見てみよう

### 例1：有名人と投機のニュースに「馴染めない」

```
人がやること自体を否定する気はないんだけど、
それなりの有名人が、若年層がよく見る媒体で、投機についての
ニュースが何度も扱われてる。どうも馴染めない。
メディアが取り上げる意図は？合理的に批判して。私の思い込み？（クリティカル・シンキングのキーワード）

URL：（記事のURL）
```

### 例2：健康・食品系の「これ本当？」

```
なんか話がうますぎて逆に怪しいと感じた記事。
「〇〇を食べると△△に効く」的なやつ。
誰が得する構造？情報源は？気にしすぎかな？（クリティカル・シンキングのキーワード）

URL：（記事のURL）
```

### 例3：政治・世論系の「なんか誘導されてる気が」

```
読み終わったあと、なんか特定の方向に気持ちを持っていかれた感じがした記事。
どこかに誘導する構造になってる？目的は？（クリティカル・シンキングのキーワード）

URL：（記事のURL）
```

### 例4：経済・統計系の「その数字、盛ってない？」

```
「急増」「過去最大」みたいな見出しにちょっと引っかかった記事。
構造的な欠陥があれば名指しで、数字の情報源は？反論は？（クリティカル・シンキングのキーワード）

URL：（記事のURL）
```

どれも共通しているのは、**冒頭に生の気持ち**、**中盤にキーワードのスパイス**、**最後に自分へのブレーキ**、という同じ骨格です。この型さえ体に入れば、あとはジャンルを問わず応用できます。

## 本当の面白さは「二の矢、三の矢」から始まる

ここが、この記事で一番お伝えしたいことかもしれません。

最初の質問で返ってきた答えは、**ゴールではなくスタート地点**です。AIの回答を読んで「あ、この部分がまさに引っかかってたんだ」とか、逆に「いや、そこじゃない気がする」と感じたら——**その感覚を、また次の質問として投げる**。ここから本番が始まります。

たとえば、さっきの投機ニュースの例。最初の答えで「若年層に射幸心を煽る構造がある」と返ってきたとします。そこで終わらせず、こう重ねてみる。

```
「射幸心を煽る」って言うけど、じゃあ普通の商品広告と何が違うの？
そこの線引きが曖昧なままだと、私も納得できない。もっと詰めて。
```

すると、「広告一般との違い」という新しい論点が立ち上がってくる。それにまた返ってきた答えを読んで、「でも、それって規制する側の理屈じゃない？」とさらに突っ込む。

こうやって**答え → 引っかかり → 次の問い**を繰り返していくうちに、面白いことが起きます。最初は「なんか馴染めない」としか言えなかったモヤモヤが、掘り進めるほどに輪郭を持ち始めて、「あ、自分が本当に嫌だったのはコレだったのか」という**違和感の核**にたどり着くんです。

大事なのは、追加質問には正解も遠慮もいらないということ。「そこじゃない」「もっと詰めて」「それって本当？」——思いついた引っかかりを、そのまま次の一手にするだけ。この往復運動そのものが、あなたの思考を深く掘り下げていってくれます。

一発で答えが出ないことを、がっかりしないでください。**むしろ、一発で終わらないからこそ価値がある**。掘るたびに、自分でも気づいていなかった自分の考えが顔を出してくる。これはAIという「無限に付き合ってくれる壁打ち相手」がいて初めてできる、贅沢な思考の遊びです。

## 使うときの、ちょっとした心得

**AIの答えにも、同じ目を向ける。** AIも間違えるし、もっともらしく嘘をつくこともあります。返ってきた答えに「本当にそう？」とツッコむところまでがワンセットです。

**最新ニュースは事実確認を。** AIの知識は古いことがあります。「これ、事実として合ってる？」は一次情報で確かめましょう。

**答えを急がない。** 一発で結論を出そうとせず、二の矢、三の矢を楽しむこと。その往復の時間こそが、一番あなたの頭を鍛えてくれます。

>　**AIの弱点**  
> ・存在しない情報をもっともらしく作る  
> ・最新情報を把握していないことがある  
> ・記事のURLを直接読めない場合がある  
> ・質問の言い方に引っ張られる  
> ・多数派の説明を、正解のように扱うことがある  
> ・情報源の独立性を十分に確認できないことがある

## おわりに：違和感は、磨けば武器になる

「なんか馴染めない」——この一言を、恥ずかしがらずにそのまま投げてみてください。整える必要も、賢い質問を用意する必要もありません。生のままで十分です。

そこにキーワードをひとつまみ、自分へのツッコミをひとさじ加えて投げる。返ってきた答えに、また引っかかったら、もう一度投げる。この繰り返しの中で、あなたの違和感はどんどん形を持ち、いつしか「自分が本当に大切にしていた価値観」の正体まで見えてきます。

そして何度も繰り返すうちに、AIなしでも「あ、これ因果と相関がすり替わってるな」と自分で気づけるようになる。それが本当のゴールです。

違和感は、放っておけばただのストレス。でも、問いを重ねて掘り下げれば、流されない自分をつくる武器になります。

## おまけ

一般に出回っている、Chatアプリの状況は掴みきっていないのですが、もし上記の方法で期待したような答えが戻ってこなかったり、煽りを増幅するような会話になるようでしたら、もう少し細かい指示が必要になると思います。

その時のためのPromptも落としておきます。
記事のURLと違和感（任意）を添えて、次の文章をそのままAIに入力してみてください。

```text
以下のニュース記事を、メディア・リテラシーとクリティカル・シンキングの教材として分析してください。

記事URL：
［ここに記事のURLを貼る］

私が感じた違和感（任意）：
［例：有名人の損失を娯楽として扱っているように感じた］

私の違和感を正当化することを目的にせず、その見方自体が妥当かどうかも含めて検討してください。記事を一方的に擁護したり非難したりせず、以下の順番で分析してください。

1．記事の要約
記事が伝えている内容を、感情的な表現を避けて簡潔にまとめてください。

2．情報の分類
記事内の情報を、次の項目に分けてください。
・確認可能な事実
・本人や関係者からの引用
・記者または媒体による解釈
・根拠が十分に示されていない推測
・価値判断や感情的表現

3．見出しと本文の関係
・見出しは本文を正確に反映しているか
・誇張、省略、ミスリードの可能性はあるか
・金額、固有名詞、感情語がクリックを促すために使われていないか
を分析してください。

4．フレーミング
この記事が出来事をどのような物語として見せているか説明してください。
例：成功物語、失敗談、復活劇、対立、炎上、悲劇、自己責任、感動物語など。

5．強調されている情報と省略されている情報
読者の判断に必要であるにもかかわらず、記事で説明されていない可能性がある情報を挙げてください。分母、比較対象、期間、条件、リスク、過去の経緯、一次資料などにも注目してください。

6．関係者とインセンティブ
本人、媒体、広告主、配信プラットフォーム、関連企業、読者など、関係者ごとに得られる利益と負うリスクを整理してください。
媒体の「意図」を断定せず、複数の可能性として示してください。

7．想定読者と影響
どのような読者を想定した記事かを考え、繰り返し接触した場合に生じ得る影響を分析してください。特に若年層への影響が考えられる場合は、模倣だけでなく、「何を普通だと思うようになるか」という観点から検討してください。

8．最も合理的な批判
この記事に対して成立する、感情論に頼らない強い批判を提示してください。

9．最も合理的な擁護
この記事を作った側に立ち、成立し得る最も強い擁護や反論を提示してください。批判側が見落としている可能性も指摘してください。

10．別の見出し
同じ事実について、次の3種類の見出しを作ってください。
・できるだけ中立的な見出し
・元の記事と同程度に注目を集める見出し
・公共的な理解を重視した見出し

11．追加確認が必要な点
記事だけでは判断できないこと、一次資料で確認すべきこと、検索して検証すべきことを一覧にしてください。
検索できる場合は、記事の公開日と情報源の日付を確認し、古い情報と新しい情報を混同しないでください。

12．暫定評価
確実に言えること、可能性が高いこと、推測にとどまることを分けてください。最後に、読者が自分で考えるための質問を5つ提示してください。

追加条件：
・記事にアクセスできない場合は、内容を推測せず、本文の貼り付けを求めてください。
・特定の人物を記事だけから病気や依存症と診断しないでください。
・媒体や記者の悪意を、証拠なしに断定しないでください。
・私の意見に迎合せず、私自身の思い込みや論理の飛躍も指摘してください。
・事実、推論、評価を明確に区別してください。
・判断の確信度を「高・中・低」で示してください。
```
