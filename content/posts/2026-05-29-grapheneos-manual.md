## 1. GrapheneOSの基本思想を理解する{#1}

GrapheneOSは「Googleなしで使えるAndroid」ではなく、 **「Googleを使う場合でも制御下に置くAndroid」** と理解するのが正確です。

| 機能 | 標準Android | GrapheneOS |
|---|---|---|
| Google Play | システム統合 | サンドボックス内で分離 |
| アプリ権限 | アプリが要求 | ユーザーが細粒度で制御 |
| センサーアクセス | 基本的に許可 | ネットワーク・センサー単位でOFF可 |
| ユーザー分離 | 限定的 | 完全分離プロファイル |
| セキュリティ更新 | メーカー依存 | 月次・独自パッチ |

**重要な認識:**
- 100%匿名は不可能、目標は「攻撃面を減らすこと」
- 設定が厳しすぎると使いにくくなり、結局使わなくなる
- プロファイル分離が最強の武器

## 2. 初期設定：まずやるべきこと{#2}

### 2-1. セットアップ直後に行う設定

**① ロック画面の強化**
```
設定 → セキュリティ → 画面ロック
```
- PINは6桁以上、パターンは使わない（指紋跡が残る）
- 生体認証は便利だが、**法的強制解除のリスク**あり。状況に応じて判断
- `「電源ボタンでロック」を有効化`：即座に生体認証を無効にできる

**② Duress PIN（緊急消去PIN）の設定**
```
設定 → セキュリティ → Duress password
```
- 強制解除を求められた際に入力すると、プロファイルのデータを消去する特殊PIN
- 通常PINとは別に設定しておく

**③ 自動再起動の設定**
```
設定 → セキュリティ → 自動再起動
```
- 推奨：**72時間**（使用感を損なわずBFU=Before First Unlock状態を維持）
- 電源OFFの状態は暗号化が最も強固。再起動によりこの状態をリセット

**④ 更新設定**
```
設定 → システム → システムアップデート
```
- 自動更新をON。GrapheneOSのセキュリティパッチは速い。これを活かす



### 2-2. 初期プロファイル（オーナーアカウント）の原則

> **オーナープロファイルには極力アプリを入れない**

オーナープロファイルはデバイス全体の管理者権限を持つため、最小限の用途に留める。

推奨オーナー用途：
- システム設定のみ
- VPNクライアント（全プロファイルに適用するため）
- 緊急用の通話アプリ

## 3. プロファイル戦略：用途別に分離する{#3}

GrapheneOSの最大の強みは**完全分離プロファイル**です。プロファイル間でアプリ・データ・通知は共有されません。

```
設定 → システム → 複数ユーザー
```

### 3-1. 推奨プロファイル構成（中級者向け）

```
┌─────────────────────────────────────┐
│  オーナープロファイル                   │
│  用途:設定・VPN管理のみ                │
│  Google：なし                        │
└─────────────────────────────────────┘
         │
┌────────┴────────┬─────────────────────┐
│                 │                     │
▼                 ▼                     ▼
プロファイルA      プロファイルB          プロファイルC
「日常用」         「Google依存用」        「使い捨て/調査用」
Signal, 銀行      Gmail, Maps           検証用アプリ
FOSS系アプリ       YouTube               捨てアカ連携
```

### 3-2. 各プロファイルの設定方針

**プロファイルA：日常用（最重要）**
- Sandboxed Google Playを入れない、またはサブセットのみ
- Signal / Element / Briar など非Google通信
- F-DroidやAuroraStore経由のアプリ
- Bitwarden（パスワード管理）

**プロファイルB：Google依存用**
- Sandboxed Google Playをここに封じ込める
- Googleアカウントとの連携はここだけ
- このプロファイルがハックされても他に波及しない

**プロファイルC：使い捨て/実験用**
- 使用後にプロファイルごと削除可能
- 怪しいAPKのテスト
- 一時的に使うサービス

### 3-3. プロファイル切り替えの利便化

- ロック画面から切り替え可能（設定でON/OFFできる）
- プロファイルAに最低限の通話機能を持たせておくと日常使いが楽

## 4. アプリ調達と管理{#4}

### 4-1. アプリストアの選択肢

| ストア | 特徴 | 推奨用途 |
|---|---|---|
| **Sandboxed Google Play** | Google Playをサンドボックス内で動作 | 必要なGoogleアプリのみ |
| **F-Droid** | オープンソースアプリのみ | 日常ツール全般 |
| **Aurora Store** | Google Playの匿名フロントエンド | Playアプリを匿名で取得 |
| **直接APK** | 公式サイトから取得 | Signal, Bitwarden等 |

### 4-2. Sandboxed Google Playの賢い使い方

GrapheneOS独自機能。Google Playをシステム権限なしで動作させる。

```
設定 → アプリ → Google Play開発者サービス
→ ネットワーク権限：必要時のみ
→ センサー：OFF
```

**重要な制限事項：**
- Googleアカウントにログインしても、デバイス識別子は標準Androidより厳しく制限される
- Push通知はGoogle経由になる（Signalのような非Google通知を使うアプリは別途設定）

### 4-3. おすすめFOSSアプリ一覧

| カテゴリ | アプリ名 | 代替するもの |
|---|---|---|
| ブラウザ | **Vanadium**（標準）/ Firefox | Chrome |
| メッセージ | **Signal** | LINE, WhatsApp |
| メール | **Thunderbird** / K-9 Mail | Gmail |
| マップ | **Organic Maps** / OsmAnd | Google Maps |
| パスワード | **Bitwarden** | Google パスワード |
| カメラ | **Secure Camera**（標準）| Google Camera |
| ファイル管理 | **Material Files** | Google Files |
| 2FA | **Aegis** | Google Authenticator |
| カレンダー | **Proton Calendar** / Simple Calendar | Google Calendar |
| クラウド | **Proton Drive** / Nextcloud | Google Drive |
| RSS | **ReadYou** | Google News |
| YouTube | **Tubular** / NewPipe | YouTube公式 |

### 4-4. アプリへの権限付与の原則

```
設定 → プライバシー → 権限マネージャー
```

**最小権限の原則：**
- 位置情報：「使用中のみ」を基本とし、常時許可は原則与えない
- 連絡先：必要なアプリのみ（SNSには基本的に不要）
- マイク・カメラ：使用するたびに確認するよう設定
- ストレージ：スコープ付きアクセスのみ

**GrapheneOS独自権限（特に重要）：**

| 権限 | 内容 | 推奨 |
|---|---|---|
| `ネットワーク` | インターネット接続のON/OFF | 不要なアプリはOFF |
| `センサー` | 加速度計・ジャイロ等 | 多くのアプリは不要 |
| `Nearby Wi-Fi networks` | 周辺Wi-Fi情報（位置特定に使われる） | 原則OFF |

## 5. ネットワーク設定{#5}

### 5-1. プライベートDNS（DoH/DoT）

```
設定 → ネットワークとインターネット → プライベートDNS
```

**推奨DNS:**
- `dns.mullvad.net`（広告・トラッカーブロック付き）
- `dns.quad9.net`（マルウェアブロック付き）
- `1dot1dot1dot1.cloudflare-dns.com`（高速・基本的なプライバシー）

ISPによるDNS監視を防ぐための最低限の設定。

### 5-2. VPNの選択と設定

**VPNプロバイダーの選定基準：**
- ノーログポリシーが第三者監査済み
- オープンソースクライアント
- WireGuardプロトコル対応

**推奨プロバイダー：**
- **Mullvad VPN**：最もプライバシー重視、現金払い対応
- **ProtonVPN**：無料プランあり、スイス拠点
- **IVPN**：監査済み、最小限のデータ収集

**設定のポイント：**
```
VPNアプリ → Kill Switch（VPN切断時に通信をブロック）を必ずON
```

オーナープロファイルでVPNを設定すると全プロファイルに適用できる（GrapheneOS独自機能）。

### 5-3. Wi-Fi設定の強化

```
設定 → ネットワーク → Wi-Fi → [ネットワーク名] → プライバシー
```
- **MACアドレスのランダム化**：接続ごとにランダムを選択
- 自動接続は信頼できるネットワークのみ
- 公衆Wi-Fi使用時は必ずVPN経由

## 6. 権限・センサー管理{#6}

### 6-1. Exploit保護（GrapheneOS独自）

```
設定 → セキュリティ → Exploit protection
```

- **Memory tagging（MTE）**：メモリ破壊攻撃を防ぐ。Pixel 8以降で利用可
- **JIT無効化**：特定アプリのJIT（Just-In-Time）コンパイルを無効化

### 6-2. センサー権限の活用

多くのユーザーが見落とす設定。広告SDKは加速度計やジャイロで「フィンガープリント」を作成できる。

```
設定 → プライバシー → センサー権限
```

- ゲームや健康アプリ以外は基本OFF
- アプリ別にON/OFFできるのはGrapheneOSの強み

### 6-3. カメラ・マイクのインジケーター

GrapheneOS（Android 12以降）ではカメラ・マイク使用中に画面右上にインジケーターが表示される。これが意図せず点灯していたら即座に確認する習慣を。

## 7. ブラウザ戦略{#7}

### 7-1. Vanadium（標準ブラウザ）

GrapheneOS標準のChromiumベースブラウザ。セキュリティパッチが速い。

**推奨設定：**
```
Vanadium設定 → プライバシーとセキュリティ
```
- サードパーティCookieをブロック
- Do Not Track：送信（効果は限定的だが無害）
- HTTPS-Onlyモードをオン

### 7-2. Firefox + uBlock Origin の組み合わせ

より高度なフィルタリングが必要な場合。

**uBlock Originの設定（中級者向け）：**
- EasyList + EasyPrivacy：標準有効
- uBlock filters – Annoyances：推奨
- 「中レベルブロッキング」：慣れたら試す（一部サイトが壊れる場合あり）

### 7-3. ブラウザ使い分け戦略

| 用途 | ブラウザ | 理由 |
|---|---|---|
| 日常閲覧 | Vanadium | 速い・安全・Google不要 |
| ログインが必要なサービス | プロファイルBのChrome | 利便性優先 |
| 匿名調査 | Tor Browser（F-Droid） | 最高匿名性 |

## 8. 通知とロック画面の設定{#8}

### 8-1. ロック画面の通知制限

```
設定 → 通知 → ロック画面上の通知
```
- 「通知の内容を表示しない」を推奨
- メッセージの内容が画面に表示されなくなる（覗き見防止）

### 8-2. 通知へのアクセス制限

```
設定 → アプリ → 特別なアプリアクセス → 通知へのアクセス
```
- 通知リーダー系のアプリには与えない
- 許可しているアプリを定期的に見直す

## 9. バックアップ戦略{#9}

### 9-1. GrapheneOSのバックアップの注意点

- **Google Oneバックアップは使わない**（Googleにデータが渡る）
- SeedVaultがGrapheneOSに統合されている

```
設定 → システム → バックアップ → SeedVault
```

**SeedVaultの特徴：**
- ローカルストレージまたはNextcloudへのE2E暗号化バックアップ
- アプリデータ・SMSを含む

### 9-2. 推奨バックアップフロー

```
SeedVault
    ↓
microSDカードまたはUSBストレージ（オフライン保管）
    ↓ 必要に応じて
Proton Drive / Nextcloud（クラウド保管）
```

### 9-3. 手動バックアップが必要なもの

- Aegis（2FA）：エクスポートして暗号化保存
- Signal：設定 → チャット → チャットのバックアップ
- Bitwarden：自動クラウド同期（ただしVaultはE2E暗号化済み）

## 10. シナリオ別おすすめ設定{#10}

### シナリオA：「普通の人より少し気にする」レベル

最低限これだけやれば効果大：
- プライベートDNS設定（Mullvad or Quad9）
- MACアドレスランダム化をON
- ロック画面の通知内容を非表示
- Signalを主要メッセンジャーに
- Bitwarden導入
- Aegis（2FA）導入
- プロファイルをA（日常）とB（Google用）の2つに分離

### シナリオB：「ジャーナリスト・活動家・ビジネス重要人物」レベル

- シナリオAの全て
- 信頼できるVPNを常時接続
- Google依存プロファイルを完全に分離、普段は使わない
- Duress PINを設定
- 自動再起動を24〜48時間に設定
- 公衆Wi-FiはVPN必須
- Torブラウザを特定用途で使用
- E2Eメール（Proton Mail）に移行

### シナリオC：「一般的な中級者」の現実的な設定

完璧を求めず、日常的に継続できる設定：
- プロファイルB（Google用）でGmailやMaps等を引き続き使う
- 日常はプロファイルAで過ごし、Google依存アプリはプロファイルBに切り替え
- VPNは「外出時のみ」でも十分
- Sandboxed Google PlayはプロファイルBに限定

## 11. やりすぎ注意：利便性を損なわない妥協点{#11}

### やりすぎると逆効果になる設定

| 設定 | リスク | 推奨妥協点 |
|---|---|---|
| 全アプリのネットワークをOFF | アプリが機能しない | 明らかに不要なもの（ゲーム等）だけOFF |
| Torを常用 | 速度が遅く、多くのサービスがブロック | 特定用途のみ |
| Google完全排除 | 地図・翻訳・決済等が使えなくなる | プロファイルBで隔離して使う |
| 全位置情報をOFF | Mapsが使えない、緊急通報で位置が伝わらない | アプリ別に「使用中のみ」で許可 |
| 生体認証を無効 | 毎回PIN入力でストレス | 普段はON、機密状況では電源ボタンでロック |

### 「プライバシー vs 利便性」の現実的なスタンス

> **完璧なプライバシーは存在しない。目標は「トレードオフを意識的に選択すること」**

- Googleを使っても、「サンドボックス内で使っている」ことに意味がある
- 全部完璧にしようとして挫折するより、70%の設定を継続する方が価値がある
- 定期的（3ヶ月ごとなど）に設定を見直すサイクルを作る

## まとめ：優先度の高い順にやること{#12}

```
優先度 高
│
├── 1. プロファイルを2〜3個に分離する
├── 2. プライベートDNSを設定する
├── 3. Signalに移行する
├── 4. Bitwarden + Aegisを導入する
├── 5. ロック画面の通知内容を非表示にする
├── 6. MACアドレスランダム化をON
├── 7. VPNを導入する（外出時少なくとも）
├── 8. アプリ権限の棚卸しをする
└── 9. Duress PIN・自動再起動を設定する
│
優先度 低
```

GrapheneOSはツールです。正しく設定することで、Googeleのエコシステムのメリットとプライバシーのメリットをどちらも享受できます。完璧を目指すのではなく、**意識的なコントロール**を手に入れることがゴールです。
